INFERENCIA DE TIPO

Quando estamos programando em um linguagem estática como Java ou C# devemos declarar os tipos das variáveis obrigatoriamente.  Em Java declaramos o tipo da variável e somente podemos atribuir valores deste tipo para esta, em C# porém, além desta forma podemos usar a palavra reservada varque no momento da atribuição o compilador irá verificar o tipo do valor que está sendo atribuído e automaticamente irá entender que aquela variável é do mesmo tipo do valor que queremos guardar.

Bem, veja o Java e o C#, eles te obrigam a declarar os tipos das variáveis. Em Java você sempre terá de declarar os tipos. Em C# você pode usar o var para inferir o tipo e deixar o seu código mais limpo mas nem todos os lugares poderá fazer isso, com variáveis que pertencem a uma classe por exemplo é obrigatória a declaração do tipo.

Linguagens como PHP, Ruby, Python, Javascript e muitas outras inferem o tipo sempre, nessas linguagens você não declara os tipos

Mas o que torna estas linguagens fortemente tipadas não é só a declaração explícita de tipo, mas sim o fato de que você não conseguirá realizar alguns tipos de operações com variáveis de tipos diferentes, como somar um número int com boolean, ou subtrair um float de uma string e por aí vai.

Já linguagens como PHP e Javascript os tipos também são inferidos porém sua tipagem é fraca o que faz com que anomalias aconteçam quando se executam operações com valores de tipos diferentes. Primeiro execute o código do arquivo tipagem_fraca.php, nele estamos somando um int com string que por coincidência tem um número no meio, e o que o interpretador faz? Ao executar você verá que ele converteu sua string para int e somou ao outro valor sem nem acusar nada. Com o arquivotipagem_fraca.js acontece a mesma coisa, aqui estamos somando um int com boolean, ele não acusa problema, ele simplesmente realiza a conversão do boolean para 1 e soma com o outro valor.
